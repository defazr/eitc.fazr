## 오늘 완료한 작업

### Canonical URL 정규화 (PR #5, Round 1)

**문제**: 네이버 웹마스터에서 광고 파라미터 URL(`?gad_source=1&gad_campaignid=...`)이 별도 색인되어 홈 노출 분산 (정상 3,236노출 vs 파라미터 458노출, 추정 24클릭 손실)

**해결**:
1. `layout.tsx`에서 `alternates.canonical: "/"` 제거 (전역 상속 방지)
2. 홈 `page.tsx`에 개별 `alternates.canonical: "/"` 추가

**오디트 발견**:
- calculator/eligibility는 `"use client"` page.tsx이지만, **라우트별 layout.tsx**에서 이미 metadata + canonical 설정됨
- faq, updates, updates/[slug]도 이미 canonical 설정 완료
- 결과적으로 6개 페이지 유형 전부 canonical 커버됨
- Round 1.5 (컴포넌트 분리) 불필요 확정

## 변경 파일

| 파일 | 변경 |
|---|---|
| `src/app/layout.tsx` | `alternates: { canonical: "/" }` 제거 (3줄 삭제) |
| `src/app/page.tsx` | `Metadata` import + `alternates: { canonical: "/" }` 추가 (7줄 추가) |
